# BitBurner Script Maintenance Prompt

Use this prompt when working on BitBurner scripts in this workspace.

## Purpose
Improve BitBurner scripts by adding or refining JSDoc comments while preserving Netscript behavior.

## What to do
- Inspect the relevant BitBurner script file.
- Add concise JSDoc comments above exported and helper functions.
- Describe function parameters clearly, including any terminal arguments handled through `ns.args`.
- Keep the `ns.args[...]` usage in the JSDoc description so the docs still explain the terminal inputs, but do not change function signatures to mirror those values.
- For terminal inputs, document the usage in the description and read them from `ns.args` inside the function body, for example:

```js
/**
 * @param {NS} ns - The Netscript API object
 */
export async function main(ns) {
    const scriptHost = ns.args[0] ?? "home";
    const targetMode = ns.args[1] ?? "ranked";
}
```

- Keep comments focused on purpose, arguments, and return values.
- Preserve BitBurner-specific conventions such as `ns.args`, `ns.tprint`, `ns.exec`, and `ns.cloud`.
- Do not change runtime behavior unless explicitly requested.

## Scope
- Apply to the current file unless the user asks for the whole project.
- Skip generated files, test-only helpers, or temporary code unless requested.

## Output
- Make the edits directly in the workspace.
- Briefly summarize what was updated.

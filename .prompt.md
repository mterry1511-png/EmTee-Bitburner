# BitBurner JSDoc Maintenance Prompt

Use this prompt when a BitBurner script in this workspace needs clearer inline documentation.

## Goal
Add or refine JSDoc for exported and helper functions while preserving Netscript behavior. If the target file has a small, obvious issue that prevents it from running as intended, fix that conservatively and note the change.

## What to do
- Inspect the target script and identify exported and helper functions.
- Add concise JSDoc above each function with:
  - a short purpose description
  - `@param` tags for inputs, including `ns` and any terminal argument usage through `ns.args`
  - `@returns` when the function returns a meaningful value
- Preserve BitBurner conventions such as `ns.args`, `ns.tprint`, `ns.exec`, `ns.cloud`, and existing script flow.
- Keep edits focused on documentation unless a minimal fix is necessary to preserve the script's intended behavior.
- Do not change logic unless the file is clearly broken and the fix is small and safe.

## Scope
- Apply to the current file unless the user asks for the whole project.
- Skip generated files, test-only helpers, or temporary code unless requested.

## Output
- Make the edits directly in the workspace.
- Briefly summarize what was updated and any small safety fixes made.
